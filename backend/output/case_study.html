Methodology

Evidence-Driven Pipeline

This research uses a six-stage pipeline where every fact is grounded in evidence, validated deterministically (no LLM reasoning in validation), escalated to browser verification only when evidence is insufficient, and finally queued for human review when confidence thresholds are not met.

Human Review Rules

  • Automated queuing: Apps are flagged for review based on deterministic rules (score < 0.90, UNKNOWN fields, insufficient evidence, browser discrepancies).
  • Evidence-driven review: The reviewer sees pipeline values alongside evidence excerpts before making a correction.
  • Audit trail: Every override records the old value, new value, and reason, enabling a full mistakes table.
  • No confidence boost: Human review does not modify confidence scores — it records ground truth corrections.
  • Sample accuracy: The pipeline's correctness is measured on the reviewed sample only, not extrapolated.

Key Design Principles

  • Evidence provenance: Every extracted value cites specific evidence IDs traceable to source URLs.
  • Deterministic validation: Python-only checks — no LLM reasoning in the validation loop.
  • Browser verification: Playwright-based checks only for disputed/UNKNOWN fields.
  • Resumability: Each app's result is cached; the pipeline can be stopped and resumed.
  • Low hallucination: The LLM is constrained to extraction only — it cannot validate or reason about correctness.
