Methodology

Evidence-Driven Pipeline

This research uses a five-stage pipeline where every fact is grounded in evidence, validated deterministically (no LLM reasoning in validation), and escalated to browser verification only when evidence is insufficient.

🔍 Discovery (Tavily) → 📄 Evidence (Firecrawl) → 🧠 Extraction (Gemini) → ✅ Validation (Python) → 🌐 Verification (Playwright)

Key Design Principles

  • Evidence provenance: Every extracted value cites specific evidence IDs traceable to source URLs.
  • Deterministic validation: Python-only checks — no LLM reasoning in the validation loop.
  • Browser verification: Playwright-based checks only for disputed/UNKNOWN fields.
  • Resumability: Each app's result is cached; the pipeline can be stopped and resumed.
  • Low hallucination: The LLM is constrained to extraction only — it cannot validate or reason about correctness.
